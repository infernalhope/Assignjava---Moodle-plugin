ObjectExplorer

ObjectExplorer je třída, která pomáhá s průběhem testů. Nabízí metody pro prozkoumání studentovy třídy. Její metody zahrnují:

  • invoke(classInstance, methodName, ... params)
  • invokeStatic(methodName, ... params)
  • invokeMethodException(Object studentClassInstance, String methodName, Object... methodParams)
  • getFieldValue(classInstance, fieldName)
  • setFieldValue(classInstance, fieldName, value)
  • sadu metod na určení modifikátoru přístupu {public, static, private, final, abstract} pro třídu, metodu i proměnnou
  • methodExists(classInstance, methodName, Object... methodParams)
  • fieldExists(classInstance, fieldName)
  • isSuccessorOf(classInstance, superClassName)
  • typesEqual(Class first, Class second)

Paramert classInstance je instance studentovy třídy, která přichází do učitelovy testovací metody jako její vstupní parametr. U metod invoke a invokeStatic je použita technika vararg neboli proměnlivého počtu parametrů. Je to z toho důvodu, aby byla zjednodušena práce v případě, kdy testovaná metoda vyžaduje více jak 1 parametr. Učitel musí tyto parametry vytvořit, naplnit a předat metodám invoke nebo invokeStatic.

Metody invoke, invokeStatic, invokeMethodException a metody methodExists a fieldExists se vždy snaží prohledávat jim poskytnutou instanci studentovy třídy. Pokud nenaleznou metodu/proměnnou v této třídě, snaží se hledání rozšířit na třídy předků.
U hledání studentových metod navíc ještě dochází k tzv. unwrappu datových typů vstupů pro studentovu metodu (objekty ve vstupním parametru methodParams). Tím je myšleno to, že instance wrappowacích tříd (Integer, Double, Boolean,...) jsou nahrazeny jejich primitivním typem. Toto se děje pouze po tom, co není studentova metoda nalezena ani ve studentově třídě ani ve třídách předků. Důvodem nutnosti převést wrappovací třídy na jejich primitivní verze je ten, že varargs primitivní typy automaticky wrappuje (zabaluje) právě do jejich wrappovacích tříd.

Navíc se u hledání studentovy metody (methodExists, invoke, atd.) postupuje tak, že se z instance studentovy třídy (a posléze i z tříd předků) získají veškeré metody a hledá se metoda s požadovaným názvem. Jakmile je taková metoda nalezena, převezmou se od metody datové typy jejich vstupních parametrů, porovnají se datové typy jejich vstupních parametrů s datovými typy parametrů poskytnutých jako vararg. V tomto procesu se jednotlivé třídy porovnávají na shodu, a to tak, že se nejprve porovná, zda jsou shodné a pokud ne, vezme se přímý předek vararg parametru a zjistí se, zda tato třída předka je shodná nebo je předkem parametru metody. Toto je potřeba z toho důvodu, že API Reflexe, při zjišťování datových typů vstupních parametrů metody (method.getParameterTypes()) vrátí datové typy určené při kompilaci, nikoliv při běhu.

Příkladem může být třeba třída java.util.Calendar, která je abstract třída, tudíž instance této třídy není možná. Instancí je vždy potomek, tedy např. třída GregorianCalendar. Uvažujme metodu setEvent(java.util.Calendar myEvent). Zavoláním method.getParameterTypes() dostaneme datový typ vstupního parametru třídu java.util.Calendar, protože ta byla přítomna při kompilaci. Pokud poté porovnáme tento vstupní parametr se vstupním parametrem ve varargs methoParams, výjde nám false, protože porovnáváme java.util.Calendar s java.util.GregorianCalendar, což nejsou stejné třídy.


Metody neslouží k testování, a proto ani nepožadují String parametr s chybovou zprávou. Pokud však dojde k vyvolání výjimky z důvodu práce se studentovou metodou, zachycení výjimky vyvolané ve studentově kódu, apod. je vytvořena chybová hláška a ta je zanesena do výsledků!


invoke, invokeStatic, invokeMethodException - Tyto metody zajistí vyvolání specifikované metody a vrátí návratovou hodnotu vyvolané metody. Metoda invokeStatic se zaměřuje na sstatické metody, takže
nepotřebuje instanci studentovy třídy a invokeMethodException vyvolá studentovu metodu a očekává, že tato metoda vyvolá výjimku, kterou posléze vrátí jako návratovou hodnotu datového typu Throwable.
isPublic, isPrivate, apod. - jsou metody, které zjistí, zda má třída, metoda nebo proměnná požadovaný modifikátor. O co, z těchto 3 uvedených, jde, určí enum TestProvider.type . 

//import z TestManager.jar
import static testmanager.TestProvider.type.*;

boolean isTrue = TestProvider.isPublic(object,"myOwnField",FIELD);

isSuccessorOf - Zjistí, zda instance implementuje nebo dědí z třídy, jejíž název se zadává jako String.
methodExists, FieldExist - otestuje, zda instance obsahuje danou metodu nebo proměnnou.
typesEqual - Jedná se o test, který rozhodne, zda jsou 2 objekty stejného datového typu.

Příklad:
@InjectInstanceOfClass(name="MyOwnClass")
public void testMethod(Object object)
{
	double vstupA = 5.22;
	double vstupB = 14.578;
	double expResult = (vstupA * vstupB) / 2;
	double result = (double)ObjectExplorer.invoke(object, "myOwnMethod", vstupA, vstupB);
	TestProvider.testEquals(expResult, result);
}


type

public enum type
{
	CLASS,
	METHOD,
	FIELD;
}

Jde o výčtový typ, který je součástí této třídy. Slouží metodám, které zjišťují jestli {metoda, proměnná, třída} má požadovaný modifikátor. Tento výčtový typ vstupuje do zmíněných metod jako formální parametr, je tedy nutné, aby si ho třída s testovacími metodami importovala.
